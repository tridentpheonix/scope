# ScopeOS Soft Launch Note

ScopeOS is open for a small pilot with real website briefs.

## What to do

1. Sign in with email/password or Google.
2. Open **New Scope Risk Check**.
3. Paste one real brief, call transcript, Loom summary, or set of notes.
4. Review the extracted scope and proposal-pack draft.
5. Use `/feedback` if anything blocks, slows, or confuses the workflow.

## What is still beta

- The product is still narrow: small agency website scoping, not a general proposal platform.
- Password reset email is not enabled yet; recovery is manual during the pilot.
- Billing is available only after Stripe checkout/webhook verification is confirmed for the current deployment.
- AI output should be reviewed by a human before it reaches a client.

## How to report problems

- Login or billing issue: open `/support` first.
- Product bug after sign-in: use `/feedback`.
- Operator review: check `/ops` and `/api/health` before debugging individual reports.
